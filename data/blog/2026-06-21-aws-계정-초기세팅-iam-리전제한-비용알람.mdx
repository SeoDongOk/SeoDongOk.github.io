집 라즈베리파이에서 돌리던 트레이딩 봇들을 AWS로 옮기기로 했다. "인프라 공부 겸"이라는 가벼운 마음으로 시작했는데, 막상 EC2를 만들기 전에 **계정 초기 세팅(보안 + 비용)**부터 제대로 하는 게 먼저였다. 이 글은 그 0단계에서 겪은 삽질과 배운 점 기록이다.

## 왜 EC2부터 안 만들고 계정 세팅부터?

처음엔 바로 EC2를 띄우고 싶었지만, 두 가지가 마음에 걸렸다.

1. **보안** — 루트 계정으로 막 작업하면 위험하고, 실수로 엉뚱한 리전에 리소스를 만들면 요금이 샌다.
2. **비용** — 실거래 봇이라 API 키가 유출되면 요금 폭탄으로 직결된다.

그래서 0단계로 **IAM 사용자 → 리전 제한 → 비용 알람** 순서로 잡았다.

## 1. 루트 대신 IAM 사용자

루트 계정은 "마스터키"라 일상 작업에 쓰면 안 된다. 권한을 가진 별도 사용자(`bot-admin`)를 만들고, 앞으로 작업은 이 사용자로 한다.

- IAM → 사용자 생성 → `AdministratorAccess` 연결 (학습 단계라 풀권한, 나중에 최소권한으로 좁힐 예정)
- 액세스 키 발급 → 로컬 `aws configure`로 CLI 연결
- `aws sts get-caller-identity`로 `.../user/bot-admin` 확인

> 삽질 ①: `aws config`라고 쳤다가 `command not found`. 명령은 `aws configure`다. 그리고 애초에 AWS CLI가 안 깔려 있어서 `brew install awscli`부터 했다.

## 2. 안 쓰는 리전 막기 — 근데 "끌 수가 없다"

콘솔 리전 목록을 보니 시드니 등 안 쓰는 리전이 잔뜩 활성화돼 있었다. 시드니부터 끄려고 했는데—

**AWS 리전은 두 종류였다.**

- **기본 활성 리전**(시드니·서울·도쿄 등) → 항상 켜져 있고 **비활성화 불가**
- **opt-in 리전**(홍콩·자카르타 등 신규) → 이건 enable/disable 가능

시드니는 기본 리전이라 "끄기" 메뉴가 아예 없다. 그래서 방향을 바꿨다: **끄는 대신, IAM 정책으로 "서울 외에는 리소스를 못 만들게" 막는다.**

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyAllOutsideSeoul",
      "Effect": "Deny",
      "NotAction": [
        "iam:*",
        "sts:*",
        "cloudfront:*",
        "route53:*",
        "support:*",
        "organizations:*",
        "budgets:*",
        "waf:*",
        "wafv2:*"
      ],
      "Resource": "*",
      "Condition": { "StringNotEquals": { "aws:RequestedRegion": "ap-northeast-2" } }
    }
  ]
}
```

핵심은 `NotAction`에 **글로벌 서비스(IAM·STS·CloudFront 등)를 빼두는 것**. 이걸 안 빼면 본인도 잠긴다. (리전 개념이 없는 서비스라 막으면 안 됨)

> 삽질 ②: 정책을 만들려고 IAM Policies 화면에서 `DenyOutsideSeoul`을 **검색**했더니 `0 matches`. 검색창은 _기존_ 정책을 찾는 용도고, 새로 만드는 건 우측 위 **Create policy** 버튼이다. 만들기 전이니 검색에 안 나오는 게 당연했다.

> 삽질 ③: `DenyOutsideSeoul`(Deny)만 사용자에 붙였더니 **서울에서조차 아무것도 안 됐다.** IAM은 **기본이 전부 Deny**라, Allow 정책이 있어야 작동한다. Deny는 "허용을 빼는" 거지 "허용을 주는" 게 아니다. 그래서 `AdministratorAccess`(Allow 전체)를 같이 붙였다. 두 개가 공존하면 **명시적 Deny > Allow > 기본 Deny** 우선순위로 → 서울은 풀권한, 다른 리전은 차단이 정확히 구현된다.

> 삽질 ④: 정책을 걸기도 전에 콘솔 리전 목록이 다 자물쇠로 보여서 "다 잠겼나?" 했는데, 알고 보니 **루트로 보고 있어서** 그랬다. **루트 계정에는 IAM Deny가 안 먹는다.** 정책은 `bot-admin` 전용이라, `bot-admin`으로 작업해야 효과가 나타난다.

> 삽질 ⑤: 계정 메뉴에 보이던 **계정 이름**을 IAM 사용자인 줄 알고 `list-attached-user-policies`를 쳤더니 `NoSuchEntity`. 알고 보니 그건 **계정 이름(Account name)**, 그냥 계정에 붙은 표시용 라벨이었다. "사용자"가 아니라 간판이라 따로 막을 대상이 아니었다.

**검증** — `bot-admin`으로 시드니에 요청을 날려봤다:

```bash
aws ec2 describe-vpcs --region ap-southeast-2
# → UnauthorizedOperation ... with an explicit deny
#   in an identity-based policy: .../policy/DenyOutsideSeoul
```

`explicit deny ... DenyOutsideSeoul` — 정책이 정확히 시드니를 막았다. 서울은 잘 되고. 의도한 그대로다.

## 3. 비용 폭탄 방지 — AWS엔 하드 캡이 없다

가장 충격이었던 사실: **"100달러 넘으면 자동으로 과금 중단"은 AWS 기본 기능에 없다.** 핸드폰 요금제처럼 칼같이 끊기는 게 아니라, 사용량 기반 과금이다. 대신 두 단계로 막는다.

- **AWS Budgets 알람** — 한도(100달러)에 **50% / 80% / 100%** 도달 시 이메일. 요금이 튀면(보통 키 유출·실수) 바로 인지.
- **Budget Action** — 초과 시 자동으로 IAM Deny 정책을 적용해 **신규 리소스 생성 차단**. 단 이미 돌던 EC2 등은 계속 과금(완전 차단 아님).
- 진짜 "0원 하드 캡"을 원하면 Budget Action + Lambda로 리소스 자동 종료까지 엮어야 한다.

> 삽질 ⑥: Budget 만들 때 `Use a template (simplified)`로 가면 알림 임계치가 **고정(85%/100%)**이라 50/80/100을 못 정한다. **Customize (advanced)**로 가야 임계치를 직접 추가할 수 있다. 그리고 `Add action` 버튼은 "알림 추가"가 아니라 자동 차단/SNS 액션을 붙이는 거라, 이메일 알림만 쓸 거면 누를 필요 없다.

봇이 월 $15~18 수준이라, $100 한도 알람이면 충분하다고 판단했다.

## 배운 점

- **EC2부터 만들지 말고 계정 위생(IAM·리전·비용)부터** 잡는 게 맞다. 나중에 리소스가 늘면 손대기 어렵다.
- **리전은 "끄는" 게 아니라 "IAM 정책으로 막는다."** 기본 리전은 비활성화가 안 된다.
- **IAM은 기본이 Deny.** Deny 정책만으로는 아무것도 안 되고, Allow가 있어야 작동한다. 그리고 **루트는 Deny를 무시**한다.
- **AWS엔 비용 하드 캡이 없다.** Budget은 "알람"이지 "차단"이 아니다. 진짜 차단은 직접 엮어야 한다.
- 삽질의 절반은 "사용자 vs 계정 vs 계정이름" 같은 **개념 혼동**에서 나왔다. 용어를 정확히 구분하니 대부분 풀렸다.

다음 단계는 드디어 **EC2를 띄우고 봇을 올리는 1단계**. 고정 IP(Elastic IP)와 거래소 API의 IP 등록, systemd 자동 재시작이 주제가 될 것 같다.
