## 문제 상황

INK React Native 앱에서 Android release build(`:app:assembleRelease`)를 실행하자 Gradle 캐시 권한, Expo Gradle plugin 누락, AsyncStorage 로컬 Maven repo 미등록, react-native-iap flavor ambiguity, Kotlin 시그니처 불일치, R8 Metaspace 부족 문제가 연쇄적으로 발생했다.

## 원인 분석

- 기본 Gradle 캐시가 `~/.gradle`을 사용해 샌드박스 환경에서 lock 파일 생성이 실패했다.
- Expo 55 bare RN 구성을 위한 `expo-autolinking-settings`, `expo-module-gradle-plugin` includeBuild 설정이 Android `settings.gradle`에 없었다.
- AsyncStorage 3.x의 `storage-android` AAR은 패키지 내부 `android/local_repo`에 포함되어 있지만 루트 Maven repository에 등록되지 않았다.
- `react-native-iap`은 `play`/`amazon` flavor를 제공하므로 앱에서 `store` dimension 선택이 필요했다.
- RN 0.84의 native bridge API nullability/Activity getter 변화와 일부 라이브러리 코드가 맞지 않았다.
- R8/lint 단계에서 기본 `MaxMetaspaceSize=512m`가 부족했다.

## 해결 방법

- `GRADLE_USER_HOME`을 프로젝트 내부로 지정하고 `npm run android:release` 스크립트를 추가했다.
- `android/settings.gradle`에 Expo autolinking/settings plugin 및 Expo module Gradle plugin includeBuild를 추가했다.
- `android/build.gradle`에 AsyncStorage 내장 local Maven repo를 추가했다.
- `android/app/build.gradle`에 `missingDimensionStrategy "store", "play"`를 추가했다.
- `android/gradle.properties`의 JVM args를 `-Xmx4096m -XX:MaxMetaspaceSize=1024m`로 상향했다.
- `scripts/patch-native-modules.js`를 추가하고 `postinstall`에서 RN 0.84 호환 임시 패치를 자동 적용하도록 했다.

## 배운 점

- React Native 최신 버전과 Expo modules를 bare RN에 혼합할 때는 Android settings/pluginManagement 설정이 특히 중요하다.
- 네이티브 라이브러리의 local Maven repo와 product flavor는 release build 전까지 드러나지 않을 수 있다.
- `node_modules` 직접 수정은 재설치 시 사라지므로 임시라도 postinstall 패치나 patch-package 형태로 영속화해야 한다.
- release build 검증은 Metro bundle 통과와 별개로 R8/lint/native compile까지 반드시 확인해야 한다.
