> Quant의 첫 질문은 단순하다. “투자해서 얼마나 벌었는가?” 하지만 CFA에서는 이 질문을 가격 변화, 현금흐름, 기간, 위험, 인플레이션까지 연결해서 본다.

## 공식 범위 확인

2027 Level I Topic Outlines의 첫 Quant 모듈은 금융자산과 금융상품의 수익률을 설명·비교·해석하는 능력을 요구한다. 또한 요구수익률, 무위험수익률, 위험프리미엄, 인플레이션을 함께 다룬다.

## 수익률의 기본 구조

가장 기본적인 보유기간수익률은 다음처럼 생각한다.

```text
Holding Period Return = (기말가격 - 기초가격 + 기간 중 현금흐름) / 기초가격
```

예를 들어 100에 산 자산이 108이 되었고 중간에 2의 배당을 받았다면 수익률은 다음이다.

```text
(108 - 100 + 2) / 100 = 10%
```

여기서 중요한 것은 수익률이 가격 변화만이 아니라 현금흐름까지 포함한다는 점이다.

## 가격수익과 인컴수익

| 구분          | 의미                     | 예시                         |
| ------------- | ------------------------ | ---------------------------- |
| Price return  | 가격 변화에서 오는 수익  | 주가 상승, 채권 가격 상승    |
| Income return | 현금흐름에서 오는 수익   | 배당, 쿠폰, 이자             |
| Total return  | 가격수익과 인컴수익의 합 | 주식 총수익률, 채권 총수익률 |

시험에서는 “가격이 올랐다”와 “총수익률이 올랐다”를 구분해야 한다. 배당이 큰 자산은 가격수익률이 낮아도 총수익률은 높을 수 있다.

## Gross return과 Net return

수익률을 계산할 때는 `1 + R` 형태의 gross return이 자주 쓰인다.

```text
Net return = R
Gross return = 1 + R
```

예를 들어 수익률이 8%라면 gross return은 1.08이다. 여러 기간 수익률을 연결할 때는 단순히 8% + 5%처럼 더하지 않고 gross return을 곱한다.

```text
2기간 누적수익률 = (1 + R1)(1 + R2) - 1
```

이 구조를 모르면 기하평균, Time-weighted return, 연율화에서 계속 헷갈린다.

## 수익률 해석의 핵심

수익률을 볼 때는 세 가지를 같이 봐야 한다.

1. **어떤 기간의 수익률인가**
2. **현금흐름이 포함되었는가**
3. **위험과 비교했을 때 충분한가**

예를 들어 1개월 5%와 1년 5%는 의미가 다르다. 또 무위험수익률이 4%인 환경에서 5% 수익률과 무위험수익률이 1%인 환경에서 5% 수익률은 다르게 해석된다.

## 오답 포인트

| 오답 포인트                      | 체크 방법                           |
| -------------------------------- | ----------------------------------- |
| 배당 또는 쿠폰을 빼먹음          | 문제에 cash flow가 있는지 먼저 표시 |
| 기간이 다른 수익률을 그대로 비교 | 연율화 여부 확인                    |
| 수익률을 단순 합산               | gross return으로 곱해서 연결        |
| 가격수익률과 총수익률 혼동       | income component 포함 여부 확인     |

## 체크리스트

- [ ] 보유기간수익률 공식을 쓸 수 있다.
- [ ] 가격수익률과 총수익률을 구분할 수 있다.
- [ ] gross return으로 여러 기간 수익률을 연결할 수 있다.
- [ ] 수익률 해석 시 기간과 현금흐름을 함께 확인한다.

수익률은 Quant의 첫 단추다. 이 단추가 어긋나면 뒤의 기하평균, 벤치마킹, 포트폴리오 수익률까지 모두 흔들린다.
