## 문제 상황

커뮤니티 글쓰기 화면에서 로그아웃 상태인데도 사진 첨부와 글쓰기 버튼이 활성화되어 있어, 사용자가 제출 단계에서만 로그인 필요 메시지를 보게 되는 문제가 있었다.

## 원인 분석

로그인 여부를 확인하는 값은 존재했지만, 실제 UI 버튼의 `disabled` 상태와 시각적 비활성화 처리가 연결되어 있지 않았다. 그래서 입력 전부터 사용자가 현재 상태를 직관적으로 알 수 없었다.

## 해결 방법

- 로그인 여부를 `isLoggedIn`으로 계산했다.
- 사진 첨부 버튼과 게시 버튼에 `disabled`를 적용했다.
- 비활성 상태에서는 opacity를 낮춰서 화면에서도 구분되도록 했다.
- 버튼 핸들러에서도 로그인 상태가 아니면 즉시 종료하도록 방어 코드를 추가했다.

## 배운 점

- 제출 시점의 에러 처리만으로는 UX가 부족하고, 버튼 레벨에서 상태를 미리 드러내는 것이 중요하다.
- 로그인/권한 상태는 기능 차단뿐 아니라 시각적 피드백까지 함께 처리해야 한다.
