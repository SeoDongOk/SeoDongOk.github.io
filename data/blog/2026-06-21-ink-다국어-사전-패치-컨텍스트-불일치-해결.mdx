## 문제 상황

영어·한국어·베트남어 사전에 비자 관리 번역 키를 한 번에 추가하는 패치가 `Failed to find expected lines` 오류로 중단되었다.

## 원인 분석

세 언어 파일의 객체 구조는 같았지만 기존 번역 문장의 표현이 언어별로 달랐다. 특히 한국어 `calculator.note` 값이 패치에서 예상한 문장과 실제 파일의 문장이 달라, 여러 파일을 묶은 패치 전체가 컨텍스트를 찾지 못했다.

## 해결 방법

각 언어 파일의 `visaManager` 블록을 먼저 조회해 실제 문맥을 확인했다. 이후 공통 구조만 기준으로 패치를 나누고, 언어별 실제 문장을 정확한 앵커로 사용해 번역 키를 추가했다.

## 배운 점

여러 로케일 파일을 동시에 수정할 때 객체 키 구조가 같아도 번역 값은 동일한 패치 앵커로 사용할 수 없다. 값보다 안정적인 키 라인을 앵커로 사용하고, 변경 전 각 로케일의 실제 블록을 확인해야 한다.
