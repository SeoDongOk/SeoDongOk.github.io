## 오늘의 코드 개선

### 1. 진입 신호 캔들 경계 제한

**문제:** `FUTURES_CHECK_INTERVAL=1m`이라 매분 실행되면서 동일한 15m 캔들에 대해 신호가 최대 15번 반복 발생.

**수정:** 포지션이 없을 때 진입 신호 생성은 15분 경계(00, 15, 30, 45분)에만 실행.

```python
def _at_candle_boundary(self) -> bool:
    tf = self.settings.timeframe  # "15m"
    now = datetime.now()
    if tf.endswith("m"):
        minutes = int(tf[:-1])
        return now.minute % minutes == 0
    ...
```

SL/TP 모니터링은 여전히 매분 수행 → 빠른 청산 대응 유지.

### 2. SDO v2 추세 기준 시간 파라미터화 + 최적값 탐색

### 3. 트레일링 SL 버그 수정 (→ 복원)

어제 트레일링 SL의 `distance_pct`에 레버리지 나누기가 빠진 버그를 수정했는데, 오늘 실제 거래에서 1분 만에 청산되는 문제 발생.

원인 파악: `.env` 주석 확인 결과 `TRAIL_ACTIVATION_PCT`, `TRAIL_DISTANCE_PCT`는 **원래부터 가격 기준 %로 의도된 설계**였음.

```
TRAIL_ACTIVATION_PCT=0.5  # 가격 +0.5% 이후 트레일링 시작
TRAIL_DISTANCE_PCT=0.6    # 고점 대비 -0.3% (촘촘한 추적)
```

`stop_loss_pct`는 마진 기준 %, 트레일링 파라미터는 가격 기준 % — 단위가 다른 것은 의도된 설계. 레버리지 나누기를 다시 제거하여 원복.
