## 문제 상황

React Native Metro 서버를 다시 띄우는 과정에서 `listen EPERM: operation not permitted 0.0.0.0:8081` 오류가 발생했다.  
기본 `npm start`는 Metro를 `0.0.0.0:8081`에 바인딩하려고 했고, 현재 실행 환경에서는 이 바인딩이 허용되지 않았다.

## 원인 분석

오류 메시지 그대로, 문제는 앱 코드가 아니라 서버가 리슨할 주소였다.  
`0.0.0.0`은 모든 인터페이스에 바인딩하는 방식인데, 이 환경에서는 해당 바인딩이 거부되었다.

즉 Metro 자체가 고장 난 것이 아니라, 기본 호스트 설정이 현재 실행 환경과 맞지 않았던 것이다.

## 해결 방법

Metro를 `127.0.0.1`에만 바인딩하도록 실행했다.

```bash
npx react-native start --host 127.0.0.1
```

이렇게 실행하니 `http://127.0.0.1:8081`에서 Metro 서버가 정상적으로 올라왔다.

## 배운 점

로컬 개발 서버가 안 뜰 때는 코드보다 먼저 바인딩 주소와 포트 충돌부터 확인하는 게 빠르다.  
특히 RN/Metro처럼 기본값이 `0.0.0.0`인 도구는 실행 환경에 따라 `127.0.0.1` 지정만으로 바로 해결되는 경우가 있다.
