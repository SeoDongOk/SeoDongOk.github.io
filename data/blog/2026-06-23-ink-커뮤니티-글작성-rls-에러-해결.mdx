## 문제 상황

커뮤니티에서 새 글을 작성할 때 게시가 실패했습니다.
에러 메시지가 앱에서는 공통 실패 문구로만 보여서 원인을 바로 알기 어려웠습니다.

## 원인 분석

- `community_posts` 테이블에 대한 Row Level Security 정책이 부족했습니다.
- 읽기 조회는 가능했지만, 작성자 기준 insert policy가 없어 글 등록이 막혔을 가능성이 높았습니다.
- 앱에서는 실제 에러 메시지를 숨기고 있어서 디버깅이 어려웠습니다.

## 해결 방법

- `community_posts`에 대해 조회/작성/수정/삭제 정책을 추가했습니다.
- insert는 `auth.uid() = author_id` 조건으로 본인 글만 등록되도록 했습니다.
- 커뮤니티 글 작성 화면에서 실패 시 실제 에러 메시지를 같이 표시하도록 바꿨습니다.

## 배운 점

- Supabase는 조회는 되는데 쓰기만 막히는 경우가 많아서 RLS 정책을 꼭 함께 확인해야 합니다.
- 에러를 무조건 공통 문구로만 숨기면, 원인 파악 시간이 길어집니다.
- 작성 기능은 insert policy와 실제 에러 노출을 같이 점검하는 게 좋습니다.
