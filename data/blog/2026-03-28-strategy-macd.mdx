## 전략 개요

**MACD(Moving Average Convergence Divergence)** 는 기술적 분석에서 가장 널리 쓰이는
모멘텀 지표 중 하나다. 두 EMA의 차이를 이용해 추세의 방향과 강도, 전환점을 파악한다.

이 전략은 단순 MACD 크로스에 **히스토그램 방향 전환** 조건을 추가해
신뢰도를 높인 버전이다.

---

## MACD 지표 구조

```
MACD 라인   = EMA(12) - EMA(26)
시그널 라인 = MACD 라인의 EMA(9)
히스토그램  = MACD 라인 - 시그널 라인
```

### 각 요소의 의미

| 요소        | 의미                               |
| ----------- | ---------------------------------- |
| MACD 라인   | 단기/장기 EMA의 차이 → 모멘텀 방향 |
| 시그널 라인 | MACD의 평활값 → 기준선 역할        |
| 히스토그램  | MACD - 시그널 → 크로스 임박 신호   |
| 제로선(0)   | MACD가 위 = 단기가 장기보다 강함   |

## 진입 로직

### LONG 조건 — 골든크로스

```python
# 1. MACD 라인이 시그널 라인을 하→상 돌파
cross_up = macd_prev <= signal_prev and macd_cur > signal_cur

# 2. 히스토그램이 음수→양수로 전환 (모멘텀 확인)
hist_turn_up = hist_prev < 0 and hist_cur > 0

# 3. (선택) 제로선 위 크로스만 허용 → 강한 상승 추세 확인
if zero_filter: macd_cur > 0
```

### SHORT 조건 — 데드크로스

```python
# 1. MACD 라인이 시그널 라인을 상→하 돌파
cross_dn = macd_prev >= signal_prev and macd_cur < signal_cur

# 2. 히스토그램이 양수→음수로 전환
hist_turn_dn = hist_prev > 0 and hist_cur < 0

# 3. (선택) 제로선 아래 크로스만
if zero_filter: macd_cur < 0
```

## 히스토그램 전환 조건의 역할

히스토그램이 전환되는 순간은 MACD 크로스보다 **한두 봉 앞서 발생**하는 경우가 많다.
히스토그램 전환을 함께 확인하면:

1. 크로스 직후 바로 되돌리는 가짜 신호 차단
2. 실제 모멘텀 방향 전환이 일어난 신호만 포착
3. 승률은 소폭 낮아지지만 손익비 개선

---

## zero_filter 옵션

```python
# zero_filter=False (기본): 제로선 아래 골든크로스도 허용
# → 반등 초기 진입 가능, 신호 많음

# zero_filter=True: 제로선 위 골든크로스만 허용
# → 강한 상승 추세 확인 후 진입, 신호 적음
```

하락장이 길었던 구간에서는 `zero_filter=False`가 유리하고,
추세장에서는 `zero_filter=True`로 MDD를 줄일 수 있다.

## 파라미터 설명

| 파라미터         | 기본값 | 설명                      |
| ---------------- | ------ | ------------------------- |
| `fast`           | 12     | 단기 EMA 기간             |
| `slow`           | 26     | 장기 EMA 기간             |
| `signal`         | 9      | 시그널 EMA 기간           |
| `zero_filter`    | False  | 제로선 위/아래 필터       |
| `hist_confirm`   | True   | 히스토그램 전환 필수 여부 |
| `strength_scale` | 0.001  | 신호 강도 스케일          |

### 파라미터 조정 가이드

```
# 표준 MACD (기본값, 가장 널리 사용)
fast=12, slow=26, signal=9

# 빠른 MACD (단기 반응, 신호 많음)
fast=6,  slow=13, signal=5

# 느린 MACD (장기 추세, 신호 적음)
fast=19, slow=39, signal=9
```

### SOLUSDT 15m, 30x 레버리지

| 기간  | 수익률       | 거래수 | 승률  | MDD   | Sharpe |
| ----- | ------------ | ------ | ----- | ----- | ------ |
| 90일  | **+87.30%**  | 413건  | 30.0% | 55.1% | 0.033  |
| 365일 | **+454.66%** | 1749건 | 30.0% | 53.7% | 0.031  |

### 승률 30%인데 어떻게 수익?

수익이 나는 구조를 이해하려면 **기대값(Expected Value)** 을 계산해야 한다.

```
평균 이익  = 마진 +30% × TP 확률
평균 손실  = 마진 -30% × SL 확률

기대값 = (0.30 × 30%) + (0.70 × -30%)
       = 9% - 21% = -12%???
```

그런데 실제로는 트레일링 스탑이 작동하고,
일부 거래는 큰 추세를 타서 TP를 여러 번 초과하는 효과가 있다.
MACD가 추세 시작점을 잘 잡으면 한 번의 큰 수익이 여러 번의 손절을 상쇄한다.

## 실거래 적용 시 주의사항

1. **거래 빈도 높음**: 하루 4~5건 → 수수료 누적 고려 필요
   - 실제 수수료 0.04% × 2(진입+청산) × 5건 = 하루 0.4% 수수료
2. **MDD 53%**: 연간 기준으로 절반 이상 자본이 줄어드는 구간 존재 → 멘탈 관리 중요
3. **`zero_filter=True`** 테스트 권장: MDD 개선 가능성 있음
