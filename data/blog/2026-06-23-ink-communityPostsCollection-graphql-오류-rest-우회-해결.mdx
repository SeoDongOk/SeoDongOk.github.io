## 문제 상황

커뮤니티 화면에서 `communityPostsCollection` 관련 GraphQL 오류가 발생했습니다.
권한은 허용했지만, Supabase GraphQL 조회가 계속 실패해 커뮤니티 피드를 불러올 수 없었습니다.

## 원인 분석

- 커뮤니티 조회가 Supabase GraphQL `/graphql/v1`에 강하게 의존하고 있었습니다.
- 실제 운영 환경에서 `communityPostsCollection` 스키마/필드 노출이 기대와 달라 오류가 발생했습니다.
- 확장 활성화 이후에도 GraphQL schema name 또는 field mapping 문제가 남아 있을 수 있었습니다.

## 해결 방법

- 커뮤니티 조회를 GraphQL 대신 Supabase REST client(`supabase.from('community_posts')`)로 전환했습니다.
- `region`, `status`, `category` 필터와 `created_at`, `is_pinned` 정렬은 REST 쿼리로 동일하게 재현했습니다.
- GraphQL 의존을 줄여 커뮤니티 피드가 스키마 노출 문제에 영향을 덜 받도록 했습니다.

## 배운 점

- Supabase에서 GraphQL이 열려 있어도, 실제 테이블/필드 노출이 기대와 다를 수 있습니다.
- 안정성이 중요한 목록 조회는 REST/Supabase client가 더 단순하고 디버깅하기 쉽습니다.
- 커뮤니티처럼 자주 보는 화면은 GraphQL 실패보다 복구 가능한 조회 경로를 먼저 확보하는 것이 좋습니다.
