### 4차 — futures_trader RLock 동시성 버그 수정 (53 → 55 pass)

**핵심 버그**: Mark Price WS worker와 `run_cycle()`이 동시에 `_check_sl_tp()` 또는 `_close_position()`을 호출하는 race condition.

**해결**:

```python

_check_sl_tp(), _close_position(), _partial_close_position()
_scale_in(), _open_position()
```

**추가 수정**: 실거래 청산 실패 시 내부 포지션을 보존하도록 변경 (기존: 잘못 삭제됨).

### 5차 — futures_client API 재시도 + WebSocket 안정화 (55 → 57 pass)
