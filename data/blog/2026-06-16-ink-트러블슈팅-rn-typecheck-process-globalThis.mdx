## 문제 상황

`vietnam-in-korea-superapp-rn`에서 Supabase 설정 누락 시 프로덕션 실행을 막기 위해 `process.env.NODE_ENV` 기반 테스트 예외 처리를 추가한 뒤 `npm run typecheck`가 실패했다.

## 원인 분석

React Native 앱의 `tsconfig`는 Node 타입을 포함하지 않기 때문에 TypeScript가 `process` 전역을 알 수 없었다. 런타임에서는 Jest가 `process.env.NODE_ENV`를 제공하지만, RN 타입 환경에서는 직접 참조하면 `TS2591` 오류가 발생한다.

## 해결 방법

`process`를 직접 참조하지 않고 `globalThis`를 좁은 타입으로 캐스팅해 테스트 환경 여부를 확인하도록 수정했다.

## 배운 점

React Native 앱 코드에서 Node 전역을 직접 참조하면 타입 환경에 따라 깨질 수 있다. 테스트 전용 분기라도 `globalThis` 기반의 안전한 접근이나 별도 런타임 헬퍼를 사용하는 편이 안정적이다.
