## 문제 상황

INK React Native 개발 서버를 `npm start -- --reset-cache`로 재시작하려 했지만, Metro가 `0.0.0.0:8081` 포트에 바인딩하는 단계에서 `listen EPERM: operation not permitted` 오류로 종료되었다. `127.0.0.1`을 명시해도 동일한 오류가 발생했다.

## 원인 분석

코드나 Metro 설정 문제가 아니라 작업 실행 샌드박스가 로컬 TCP 포트 리스닝을 제한하고 있었다. 따라서 샌드박스 내부 프로세스는 8081 포트를 열 수 없었지만, macOS 호스트 Terminal에서 실행한 프로세스는 정상적으로 포트를 사용할 수 있었다.

## 해결 방법

AppleScript로 호스트 Terminal을 열어 프로젝트 디렉터리에서 Metro를 실행했다.

```bash
cd /Users/seodong-og/project_vs/plan_ink_superapp/vietnam-in-korea-superapp-rn
npm start -- --reset-cache
```

이후 `lsof -nP -iTCP:8081 -sTCP:LISTEN`으로 Node 프로세스가 8081 포트를 정상적으로 리스닝하는 것을 확인했다.

## 배운 점

`EPERM`은 포트 충돌을 뜻하는 `EADDRINUSE`와 다르다. 포트가 비어 있어도 실행 환경의 네트워크 권한 때문에 발생할 수 있다. 샌드박스 환경에서 개발 서버가 포트를 열지 못하면 호스트 터미널에서 실행하고, 실제 리스닝 여부를 별도로 검증해야 한다.
