> 가설검정은 “표본에서 관찰한 차이가 우연인지, 의미 있는 신호인지”를 판단하는 절차다. CFA Quant에서는 공식보다 절차와 해석을 정확히 아는 것이 중요하다.

## 공식 범위 확인

2027 Quant의 Estimation and Hypothesis Testing 모듈은 가설검정과 그 구성요소, 통계적 유의성, 검정 구성과 결과 해석을 포함한다.

## 가설검정의 목적

예를 들어 어떤 펀드의 평균 초과수익률이 0보다 큰지 알고 싶다고 하자. 표본에서는 2%가 나왔지만, 이것이 실력인지 우연인지 판단해야 한다. 가설검정은 이 판단을 체계화한다.

## 귀무가설과 대립가설

| 구분                       | 의미                          |
| -------------------------- | ----------------------------- |
| Null hypothesis, H0        | 기본적으로 유지하는 가설      |
| Alternative hypothesis, Ha | 검정으로 뒷받침하고 싶은 가설 |

예시:

```text
H0: 평균 초과수익률 = 0
Ha: 평균 초과수익률 > 0
```

검정은 대립가설을 직접 증명하는 것이 아니라, 귀무가설을 기각할 충분한 증거가 있는지 보는 절차다.

## 유의수준

유의수준 `α`는 귀무가설이 참인데도 기각할 확률의 허용치다. 보통 5%가 많이 쓰인다.

```text
α = 0.05
```

유의수준을 낮추면 잘못 기각할 가능성은 줄지만, 실제 효과를 놓칠 가능성은 커질 수 있다.

## 검정통계량

검정통계량은 표본 결과가 귀무가설 기준에서 얼마나 떨어져 있는지 표준화한 값이다.

일반 구조는 다음과 같다.

```text
Test statistic = (sample statistic - hypothesized value) / standard error
```

이 값이 충분히 크거나 작으면 귀무가설을 기각한다.

## p-value

p-value는 귀무가설이 참이라고 가정했을 때, 관찰된 결과만큼 극단적인 결과가 나올 확률이다.

```text
p-value < α → H0 기각
p-value ≥ α → H0 기각하지 않음
```

주의할 점은 “H0를 기각하지 않는다”가 “H0가 참임을 증명한다”는 뜻은 아니라는 것이다.

## 단측검정과 양측검정

| 검정         | 대립가설 예시 | 사용 상황             |
| ------------ | ------------- | --------------------- |
| Upper-tailed | Ha: μ > μ0    | 평균이 더 큰지 확인   |
| Lower-tailed | Ha: μ < μ0    | 평균이 더 작은지 확인 |
| Two-tailed   | Ha: μ ≠ μ0    | 차이가 있는지 확인    |

문제에서 “greater than”, “less than”, “different from” 같은 표현을 먼저 확인해야 한다.

## 오답 포인트

| 오답 포인트                   | 체크 방법                        |
| ----------------------------- | -------------------------------- |
| H0와 Ha를 반대로 설정         | 검정하려는 방향을 문장으로 쓰기  |
| p-value 해석 오류             | 귀무가설 참 가정하의 극단값 확률 |
| 기각하지 않음을 채택으로 표현 | “fail to reject”가 정확한 표현   |
| 단측/양측 혼동                | 대립가설의 부등호 확인           |
