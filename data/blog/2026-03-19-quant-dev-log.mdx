## 🔍 왜 이렇게 갈리나

두 전략은 근본적으로 **시장을 보는 관점**이 다르다.

**VWAP Reversion (역추세)**
→ "24h 평균(VWAP)에서 너무 멀어지면 돌아온다"
→ RSI 18~19 극도 과매도 = 공포 극값 → 반등 기대
→ 지금은 계속 떨어지고 있어서 손실 중

**Supertrend EMA (추세추종)**
→ "ATR 기반 추세선이 하락으로 바뀌었고 EMA도 데드크로스"
→ 추세를 믿고 방향을 따라간다
→ 시장이 하락하면서 수익 중

이 두 가지 상충하는 관점 중 어느 쪽이 "이 시장"에 맞는지가 실험의 핵심이다.

## 오늘 할 것: Supabase 백테스팅

24시간 모의거래로는 샘플이 너무 적다. 통계적으로 유의미하려면 **최소 100거래** 이상이 필요하다.

그래서 오늘은 **Supabase 기반 백테스팅 시스템**을 만든다.

### 설계

```
Supabase PostgreSQL
  ├─ candles          — 바이낸스 히스토리컬 OHLCV 캐시
  ├─ backtest_runs    — 백테스트 실행 메타데이터
  ├─ backtest_results — 전략별 성과 (수익률, 승률, MDD, Sharpe)
  └─ backtest_trades  — 개별 거래 기록
```

### 왜 Supabase?

- PostgreSQL 기반 → 강력한 쿼리 (윈도우 함수, 집계)
- 무료 티어로 충분
- REST API + Python SDK
- 결과를 대시보드로 시각화하기 쉬움
- 여러 기기에서 같은 데이터 접근 가능

### 백테스트 계획

- 기간: 최근 90일
- 심볼: BTCUSDT, ETHUSDT
- 타임프레임: 15m
- 5전략 동시 실행 → Supabase에 결과 저장 → 리포트
