## 문제 상황

`npx react-native run-ios`로 iOS 시뮬레이터 빌드를 돌리던 중 아래 오류로 멈췄다.

```text
'ExpoModulesCore/ExpoModulesCore.h' file not found
could not build Objective-C module 'Expo'
```

그 전에 `pod install` 로그에서도 이미 힌트가 나왔다.

```text
Your project includes the Expo package, but React Native Community CLI is unable to install the related Pods.
Make sure to require autolinking scripts from Expo and call `use_expo_modules!` in your target.
```

## 원인 분석

프로젝트는 `expo` 패키지를 의존성으로 가지고 있었지만, `ios/Podfile`에는 Expo 모듈 autolinking 설정이 없었다.

즉 CocoaPods는 설치됐지만, Expo native module이 Pod 그래프에 제대로 연결되지 않았다.

문제가 있던 Podfile 상태:

- `react_native_pods.rb`만 require
- target 안에 `use_expo_modules!` 없음

이 상태에서는 `ExpoModulesCore` 헤더를 찾지 못하고 네이티브 컴파일이 깨질 수 있다.

## 해결 방법

`ios/Podfile` 상단에 Expo autolinking 스크립트를 추가하고, target 안에 `use_expo_modules!`를 넣었다.

```rb
require File.join(
  File.dirname(`node --print "require.resolve('expo/package.json')"`),
  'scripts/autolinking'
)
```

```rb
target 'INK' do
  use_expo_modules!
  config = use_native_modules!
```

그 후 다시:

```bash
cd ios
pod install
```

을 실행해 Pod 구성을 재생성했다.

## 배운 점

React Native Community CLI 프로젝트라도 `expo` 패키지를 의존성에 넣었다면 Podfile도 그에 맞춰야 한다.

정리하면 체크포인트는 이렇다.

1. `expo` 패키지가 있으면 Podfile에 Expo autolinking require 추가
2. target 안에 `use_expo_modules!` 호출
3. 변경 후 `pod install` 재실행

이번 케이스처럼 Metro/JS 문제를 다 정리한 뒤에도 네이티브 빌드가 깨질 수 있으니, iOS 실행 검증은 항상 `Podfile -> pod install -> run-ios` 순서로 보는 편이 안전하다.
