## 문제 상황

`vietnam-in-korea-superapp-rn` Edge Function 하드닝 중 여러 파일의 끝부분을 확인하려고 `tail`을 사용했지만 zsh 세션에서 `command not found: tail`이 발생했다.

## 원인 분석

작업 세션의 `PATH`가 일반 macOS 유틸리티 경로를 안정적으로 포함하지 않는 상태로 보였다. 파일 자체 문제는 아니었고, 검증 명령이 셸 환경에 의존한 것이 원인이었다.

## 해결 방법

`tail` 대신 절대경로인 `/usr/bin/tail`을 호출해 동일한 파일 검증을 계속했다. 이후 TypeScript `transpileModule` 기반 구문 확인, RN typecheck/test/lint로 변경 내용을 검증했다.

## 배운 점

자동화/에이전트 세션에서는 기본 Unix 명령도 `PATH`에 따라 실패할 수 있다. 검증 흐름이 막히면 시스템 유틸리티의 절대경로를 사용하거나, Node/Python 기반 대체 검증으로 전환하는 것이 안전하다.
