## 문제 상황

INK 앱은 화면별로 카드 폭, 패딩, 그리드 규칙이 제각각이라서 기기 크기가 달라지면 레이아웃이 쉽게 답답해 보였다. 특히 홈 화면의 요약 카드와 액션 카드가 고정 폭에 가까운 형태로 배치되어 작은 화면에서는 답답하고, 큰 화면에서는 여백이 과해 보였다.

또한 다국어 설정은 적용되어 있었지만, 사용자가 처음 진입하는 온보딩 단계에서 언어 선택이 충분히 눈에 띄는지 다시 확인할 필요가 있었다.

## 원인 분석

문제의 핵심은 공통 컨테이너와 개별 섹션의 폭 기준이 분리되어 있지 않았다는 점이었다. 일부 컴포넌트는 화면 전체 너비를 기준으로 계산하고, 다른 컴포넌트는 스타일의 minWidth에 의존해서 렌더링하고 있었다. 이 때문에 실제 콘텐츠 영역과 카드 계산값이 어긋나는 경우가 생겼다.

또한 홈 화면의 그리드 카드가 스타일만으로 2열 배치를 기대하고 있어, 화면 폭이 변할 때 안정적으로 재배치되지 않았다.

## 해결 방법

공통 `ScreenContainer`를 기준으로 반응형 여백과 최대 콘텐츠 폭을 정리했다. 화면이 넓어져도 콘텐츠가 과하게 퍼지지 않도록 하고, 모바일에서는 더 편안한 좌우 여백을 유지하도록 조정했다.

홈 화면의 핵심 카드 묶음은 실제 윈도우 너비를 기준으로 카드 폭을 계산하도록 바꿨다.

- 메트릭 카드: 화면 폭에 따라 1열/2열을 적용
- 액션 카드: 동일한 계산 방식으로 폭 통일
- 기존 `minWidth` 중심 스타일은 제거해 카드가 폭 계산을 방해하지 않도록 정리

추가로 온보딩 화면의 언어 선택 영역이 초기 진입 단계에서 유지되는지 확인했다. 현재 구조에서는 언어 선택이 온보딩 초반에 노출되도록 구성되어 있어, 다국어 진입점 자체는 유지되고 있었다.

## 배운 점

반응형 UI는 개별 화면을 예쁘게 만드는 것보다, 먼저 공통 폭 기준과 간격 규칙을 통일하는 것이 중요했다. 컨테이너 기준이 정리되면 화면별 수정량이 크게 줄고, 이후 세부 카드 디자인도 훨씬 안정적으로 맞출 수 있다.

또한 다국어 기능은 "설정이 존재한다"보다 "사용자가 처음 만나는 순간에 잘 보인다"가 더 중요했다. 온보딩 단계에서 언어 선택을 다시 확인하는 작업이 필요한 이유도 여기에 있다.
