## 1. Greed 모드 — 동적 TP 확장

### 핵심 아이디어

시장이 강하게 한 방향으로 움직일 때, 기존 TP(10%)를 자동으로 확장하여 수익을 더 크게 가져간다.

### 4가지 지표 기반 점수 체계 (최대 5점)

```
+1  볼린저 밴드 확장 중 (bandwidth 현재 > 최근 5봉 평균)
+1~2 RSI 모멘텀 (LONG: 60~75=+1, 75+=+2 / SHORT: 25~40=+1, <25=+2)
+1  MACD 히스토그램 방향 일치 & 증가 중
+1  거래량 폭증 (현재 > 20봉 평균 × 1.5배)
```

### 점수 → 레벨 → TP 매핑

| 점수  | Greed Level | TP (마진 기준)      |
| ----- | ----------- | ------------------- |
| 0점   | Lv1 (기본)  | 10% (설정값 그대로) |
| 1~2점 | Lv2         | 30%                 |
| 3~4점 | Lv3         | 50%                 |
| 5점   | Lv4         | 100%                |

### 구현 포인트

```python
def _calc_greed_level(self, df: pd.DataFrame, side: str) -> tuple[int, list[str]]:
    """매 사이클마다 시장 상태를 분석하여 Greed 레벨 결정"""
    score = 0
    # 볼린저 밴드 확장, RSI, MACD, 거래량 각각 체크
    # ...
    if score >= 5: return 4
    elif score >= 3: return 3
    elif score >= 1: return 2
    else: return 1
```

포지션 보유 중 `_check_sl_tp` 호출 시 매번 레벨을 재계산하고, 변경되면 TP를 동적으로 업데이트한다.
**레벨 하락도 허용** — 모멘텀이 약해지면 TP를 다시 낮춰서 수익 확보를 우선시한다.

## 2. 추격매매 — Greed 기반 자동 진입

기존 전략이 HOLD를 내도, **Greed 레벨이 충분히 높으면** 추세 방향으로 자동 진입한다.

### 진입 조건

1. 전략 신호가 HOLD
2. EMA(20) 기울기 + 현재가 위치로 추세 방향 결정
   - EMA 상승 + 가격 > EMA → LONG
   - EMA 하락 + 가격 < EMA → SHORT
3. 해당 방향의 Greed 레벨이 `GREED_CHASE_MIN_LEVEL` (기본 3) 이상

```python
# 횡보장 안전장치: EMA 방향이 불분명하면 None → 추격매매 차단
def _calc_chase_direction(self, df: pd.DataFrame) -> Optional[str]:
    ema20 = TechnicalIndicators.ema(close, 20)
    if curr_ema > prev_ema and current_price > curr_ema:
        return "LONG"
    if curr_ema < prev_ema and current_price < curr_ema:
        return "SHORT"
    return None  # 방향 불분명 → 진입 안 함
```

테스트 결과:

- **강세장**: LONG 감지 → Greed Lv3 → 추격매매 발동
- **횡보장**: Lv2에 그침 → 최소 Lv3 미달 → 차단

## 3. 분할매수 중 SL 선청산 버그 수정

### 문제 상황

분할매수(Scale-In) 구조: 1트랜치(25%) → 2트랜치(25%) → 3트랜치(50%)

가격이 급락할 때:

```
기존 순서:
  1. _check_sl_tp → SL(-20%) 터치 → 즉시 청산 & return
  2. 분할매수 체크 → 실행 안 됨! (이미 청산됨)
```

2트랜치(-10%)나 3트랜치(-25%) 트리거가 발동해야 하는데,
SL이 먼저 체크되면서 물타기 없이 바로 손절해버렸다.

### 해결

```
수정 후 순서:
  1. 분할매수 체크 (트랜치 남아있으면 물타기 우선)
     → 분할매수 실행 → SL/TP 스킵 (평균가 변경됨, 다음 사이클에서 새 기준 판단)
     → 분할매수 없음 → SL/TP 정상 체크
```

**트랜치가 남아있는 한 SL 대신 물타기로 대응.** 3트랜치까지 모두 소진한 후에야 SL 청산을 허용한다.

## 환경변수 요약

```bash
# Greed 모드
GREED_ENABLED=True          # 활성화
GREED_TP_LEVEL2=30.0        # Lv2 TP (마진 기준 %)
GREED_TP_LEVEL3=50.0        # Lv3 TP
GREED_TP_LEVEL4=100.0       # Lv4 TP
GREED_VOLUME_MULT=1.5       # 거래량 폭증 판정 배수
GREED_CHASE_MIN_LEVEL=3     # 추격매매 최소 레벨 (0=비활성화)
```
